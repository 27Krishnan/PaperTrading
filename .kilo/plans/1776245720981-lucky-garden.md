# Plan: Fix Image Parsing, Angel One Connection & Option Chain Issues

## Problem Summary

1. **EasyOCR Error**: "Cannot read image (this model does not support image input)"
2. **Angel One Disconnecting**: Shows connected then disconnects frequently
3. **Option Chain Not Loading**: Depends on Angel One connection
4. **LTP Not Showing**: Dashboard not displaying live prices

---

## Root Cause Analysis

### 1. EasyOCR Issue
- EasyOCR requires `torch` + `torchvision` + `easyocr` packages
- First load takes 30+ seconds (model download)
- On cloud server, these packages may not be installed

### 2. Angel One Connection
- API session expires frequently (Angel One tokens are short-lived)
- No automatic reconnection with backoff

### 3. Option Chain
- Needs Angel One API connection + cached instrument master
- Market hours: NSE 9:15 AM - 3:30 PM IST, MCX 9:00 AM - 11:30 PM IST

---

## Implementation Plan

### Step 1: Install Required Packages on Server
Run on the server where the app is deployed:
```bash
pip install torch==2.2.2 torchvision==0.17.2 easyocr==1.7.2
```

### Step 2: Improve Angel One Reconnection (DONE)
- Added `_try_reconnect()` method with 3 retry attempts
- Exponential backoff: 2s, 4s, 8s delay between attempts

### Step 3: Add Health Check Endpoint
- Create `/api/health` endpoint to check all dependencies
- Return status of: EasyOCR, Angel One, Database, Market Feed

### Step 4: Add Startup Health Check
- On app startup, verify EasyOCR loads successfully
- Pre-cache instrument master for option chain
- Report any missing dependencies

### Step 5: Market Hours Logic Fix
- Add proper market open/close detection
- Show "Market Closed" status in dashboard instead of error
